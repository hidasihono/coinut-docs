## Customer Model

Customers represent end users who deposit or withdraw through your integration. Every customer must pass KYC verification before transacting.

### Customer Statuses

| Status | Description |
|--------|-------------|
| `PENDING`  | Created, awaiting KYC document submission |
| `APPROVED` | KYC approved — eligible for transactions |
| `REJECTED` | KYC rejected — cannot transact |
| `DELETED`  | Soft-deleted — no longer active |

### Customer Types

| Type | Description |
|------|-------------|
| `DEPOSITOR` | Customer who deposits crypto (onramp) |
| `PAYEE` | Customer who receives fiat payouts (offramp) |

<Note>
  Customers go through Sumsub for KYC. After creation, redirect the user to `sumsubLink` to complete document upload.
</Note>

## POST /customer/create

Create a new customer record and initiate the KYC flow.

### Request Body

<ParamField body="name" type="string" required>
  Full legal name of the customer.
</ParamField>

<ParamField body="email" type="string" required>
  Email address. Must be unique across your partner account.
</ParamField>

<ParamField body="type" type="string" required>
  Customer type. Enum: `DEPOSITOR`, `PAYEE`.
</ParamField>

<ParamField body="isIndividual" type="boolean" optional>
  `true` for individual, `false` for business entity.
</ParamField>

<ParamField body="sumsubShareToken" type="string" optional>
  Pre-existing Sumsub share token. If omitted, a new KYC session is created.
</ParamField>

<ParamField body="externalId" type="string" optional>
  Your system's unique identifier for this customer. Use this to link Coinut records with your internal database.
</ParamField>

### Response

<ResponseField name="data.id" type="string" required>
  Unique customer UUID generated by Coinut.
</ResponseField>

<ResponseField name="data.status" type="string" required>
  Initial status. Always `PENDING` after creation.
</ResponseField>

<ResponseField name="data.sumsubLink" type="string" required>
  URL to redirect the customer to for KYC document upload.
</ResponseField>

<CodeGroup>
```bash cURL
curl -X POST "https://ramp.hexarails.ai/customer/create" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $API_KEY" \
  -H "X-Timestamp: $TIMESTAMP" \
  -H "X-Nonce: $NONCE" \
  -H "X-Signature: $SIGNATURE" \
  -d '{
    "name": "John Doe",
    "email": "john@example.com",
    "type": "DEPOSITOR",
    "isIndividual": true,
    "sumsubShareToken": "token123",
    "externalId": "partner-customer-001"
  }'
```

## GET /customer/detail

Retrieve a single customer's full profile.

### Query Parameters

<ParamField query="id" type="string">
  Customer UUID (returned from `/customer/create`).
</ParamField>

<ParamField query="email" type="string">
  Customer email address.
</ParamField>

<Warning>
  At least one of `id` or `email` is required. Providing both is allowed.
</Warning>

### Response

<ResponseField name="data.id" type="string" required>
  Customer UUID.
</ResponseField>

<ResponseField name="data.realName" type="string" required>
  Verified legal name from KYC.
</ResponseField>

<ResponseField name="data.country" type="string" required>
  ISO-3166-1 alpha-2 country code.
</ResponseField>

<ResponseField name="data.countryName" type="string">
  Human-readable country name.
</ResponseField>

<ResponseField name="data.nationality" type="string">
  Nationality code.
</ResponseField>

<ResponseField name="data.city" type="string">
  City of residence.
</ResponseField>

<ResponseField name="data.address" type="string">
  Street address.
</ResponseField>

<ResponseField name="data.dob" type="string">
  Date of birth in `YYYY-MM-DD` format.
</ResponseField>

<ResponseField name="data.isIndividual" type="boolean" required>
  Whether the customer is an individual (vs. business).
</ResponseField>

<ResponseField name="data.status" type="string" required>
  One of: `PENDING`, `APPROVED`, `REJECTED`, `DELETED`.
</ResponseField>

<ResponseField name="data.remark" type="string">
  Internal notes or rejection reason.
</ResponseField>

<ResponseField name="data.type" type="string" required>
  One of: `DEPOSITOR`, `PAYEE`.
</ResponseField>

<ResponseField name="data.externalId" type="string">
  Your system's reference ID.
</ResponseField>

<ResponseField name="data.createTime" type="string" required>
  ISO-8601 creation timestamp.
</ResponseField>

<ResponseField name="data.updateTime" type="string" required>
  ISO-8601 last update timestamp.
</ResponseField>

## GET /customer/list

List all customers under your partner account with pagination and filtering.

### Query Parameters

<ParamField query="pageNum" type="integer" default="1">
  Page number (1-based).
</ParamField>

<ParamField query="pageSize" type="integer" default="10">
  Items per page. Maximum: 100.
</ParamField>

<ParamField query="status" type="string">
  Filter by customer status. Enum: `PENDING`, `APPROVED`, `REJECTED`, `DELETED`.
</ParamField>

<ParamField query="type" type="string">
  Filter by customer type. Enum: `DEPOSITOR`, `PAYEE`.
</ParamField>

<ParamField query="externalId" type="string">
  Filter by your system's reference ID.
</ParamField>

### Response

<ResponseField name="data.list" type="object[]" required>
  Array of customer objects (same schema as `/customer/detail`).
</ResponseField>

<ResponseField name="data.pageNum" type="integer" required>
  Current page number.
</ResponseField>

<ResponseField name="data.pageSize" type="integer" required>
  Items per page.
</ResponseField>

<ResponseField name="data.total" type="integer" required>
  Total matching records.
</ResponseField>

## KYC Workflow

New customers must complete identity verification through Sumsub before they can transact.

<Steps>
  <Step title="Create customer">
    Call `POST /customer/create` with the customer's details. Save the returned `id`.
  </Step>
  <Step title="Redirect to Sumsub">
    Send the customer to the `sumsubLink` URL to upload identity documents.
  </Step>
  <Step title="Wait for webhook">
    Subscribe to `CUSTOMER_APPROVED` and `CUSTOMER_REJECTED` events. Coinut sends a webhook when KYC is decided.
  </Step>
  <Step title="Check status">
    Optionally poll `GET /customer/detail` to confirm the `status` field is `APPROVED`.
  </Step>
  <Step title="Begin transactions">
    Only proceed with deposits, trades, or payouts for `APPROVED` customers.
  </Step>
</Steps>

<Note>
  The Sumsub link expires after 7 days. If it expires, contact support to regenerate it.
</Note>
